## Known Issues / Tech Debt

### Critical (blocking v1.0.0)

| Issue | Location | Impact |
|-------|----------|--------|
| **No tests** | Entire codebase | Zero test coverage, all changes are risky |
| **Tight coupling** | `project-spec-wizard.ts` | 1500+ line monolith, untestable |
| **No error recovery** | `worker-executor.ts` | Crashes lose task progress |
| **Hardcoded delays** | `sidebar-commands.ts` | 2s spawn delay is arbitrary |

### High Priority

| Issue | Location | Impact |
|-------|----------|--------|
| GUI logic mixed with business logic | All views | Cannot test without VS Code |
| No dependency injection | All core modules | Hard to mock, hard to test |
| XML parsing scattered | Multiple files | Inconsistent error handling |
| No validation layer | Wizard inputs | Bad data propagates silently |

### Medium Priority

| Issue | Location | Impact |
|-------|----------|--------|
| Hardcoded paths | Various | Windows-only assumptions |
| No logging strategy | `logger.ts` | Inconsistent log levels |
| No configuration schema | Settings | Invalid configs accepted |
| Adapter loading is sync | `adapter-loader.ts` | Blocks UI on large adapters |

### Low Priority / Nice to Have

| Issue | Location | Impact |
|-------|----------|--------|
| No i18n | All UI strings | English only |
| No progress persistence | Executor | Can't resume after crash |
| No task priority | `task-manager.ts` | FIFO only |
| No worker affinity | Spawning | Can't pin tasks to workers |
